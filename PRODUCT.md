# Product

## Register

product

## Users

Pandar is for operators who run self-hosted Bambu printer infrastructure and need tenant-scoped visibility over users, local agents, and machine links.

## Product Purpose

Pandar replaces the cloud dependency in Bambu Studio workflows with a self-hosted hub, local agents, and a web UI. Success means an operator can understand hub health, linked agents, tenants, and printer connectivity without relying on the vendor cloud.

## Brand Personality

Calm, technical, trustworthy. The interface should feel like infrastructure software for repeated operational use, not a marketing page.

## Anti-references

Avoid decorative SaaS landing-page layouts, oversized hero content inside the app, playful consumer-device styling, and visual patterns that obscure operational state.

## Design Principles

- Keep operational state scannable.
- Prefer familiar controls over novel affordances.
- Use restrained color for navigation, status, and primary actions.
- Make tenant and agent boundaries explicit.
- Keep copy concrete and action-oriented.

## Accessibility & Inclusion

Target WCAG 2.2 AA for product UI. Do not rely on color alone for state, keep keyboard navigation visible, and respect reduced-motion preferences.
